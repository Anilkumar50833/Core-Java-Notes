> Standalone app 
---------------
A normal java program which contains main method is called standalone application.
ex:
	class Test
	{
		public static void main(String[] args)
		{
			System.out.println("Hello World");
		}
	}

> Desktop app 
----------------
It is a software application which is specially designed to perform perticular task.

It is standlone to one computer only.

ex:
	control panel
	recycle bin 
	VLC media player 
	and etc.

> Two-tier app 
--------------
Having more then one tier is called two-tier application.

Diagram: java3.1
Diagram: java3.2


> Distributed App 
-----------------
In client/server architecture, if multiple clients are giving the request to main server then main server will distribute requests to it's parallel servers to reduce the burdun of main server such type of application is called distributed application.
  
Diagram : java3.3


> Enterprises App 
------------------
An application which deals with large business complex logic with the help of middleware services is called enterprise application.

Here middleware services means authentication, autherization, security, firewall, malware production and etc.

ex:
	Facebook
	E-commerce websites 	
	
> ERP App 
-----------
ERP stands for Enterprise Resource Planning.

ERP is used to maintain the data in Enterprise.

Diagram: java3.4


> N-Tier App
-------------
Having more then two tier is called N-tier application.

Diagram: java3.5

> Applets 
-----------
It is a compiled java class which is used to send over the network as web pages.

If we need a web page with performance then we need to use html.

If we need  a web page with security then we need to use applets.

> Mobile App
----------
It is a software application or a program which is designed for wireless network devices like phone,tab,cell,cellular and etc rather then laptop's and PC's.
ex:
	Gpay
	PhonePay
	PayTM
	and etc
